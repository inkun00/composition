# 초등학생용 작곡 웹앱 PRD

- 문서 상태: Draft v1.0
- 작성일: 2026-07-04
- 가칭: 마음멜로디
- 대상: 초등학생, 교사, 보호자
- 플랫폼: 태블릿·데스크톱 우선 반응형 웹

## 1. 제품 한 줄 정의

화성학을 몰라도 아이가 분위기, 박자, 한 마디 가락을 차례로 듣고 고르면 8~16마디의 노래를 만들고, 악기와 가사를 입혀 악보·PDF·공유 링크로 완성할 수 있는 안전한 블록형 작곡 웹앱.

## 2. 문제와 제품 원칙

기존 악보 편집기는 음표와 화성 지식을 먼저 요구해 초등학생에게 진입 장벽이 높다. 이 제품은 이론을 화면 앞에 세우지 않고, 이론에 맞게 검수된 선택지만 제공한다.

1. 먼저 듣고, 다음에 고른다.
2. 음악적으로 안전한 기본값을 제공하되 언제든 세부 편집할 수 있다.
3. 오류는 “틀림”이 아니라 “이 마디에 반 박이 더 필요해요”처럼 고칠 방법과 함께 알려준다.
4. 아이 화면에는 로마숫자 화음·전문용어를 기본적으로 숨긴다.
5. 공유는 기본 비공개이며, 공개·리메이크에는 원작자 계보와 아동 안전 장치를 둔다.

## 3. 목표와 비목표

### 3.1 목표

- 첫 이용자의 80% 이상이 도움 없이 15분 안에 8마디 곡을 완성한다.
- 선택 가능한 4마디 작은악절을 출시 시점에 100개 제공한다.
- 2/4, 3/4, 4/4, 6/8, 9/8, 12/8박자를 지원한다.
- 모든 제시 가락은 해당 마디의 화음과 박자 검증을 통과한다.
- 한 마디마다 최소 24개의 서로 구별되는 가락 후보를 제공한다.
- 완성곡은 재생, 저장, A4 PDF 출력, 링크 공유, 출처가 보존되는 리메이크가 가능하다.

### 3.2 비목표

- 1차 출시에서 전문 DAW 수준의 다중 트랙 믹싱, 마이크 녹음, 실시간 공동 편집은 만들지 않는다.
- 기능화성을 모든 음악 문화의 보편 규칙으로 가르치지 않는다.
- 국악기를 서양 평균율 음색으로 재현한 결과를 전통 연주법 교육으로 표현하지 않는다.
- 공개 댓글, 1:1 메시지, 팔로워 수 경쟁 기능은 제공하지 않는다.

### 3.3 명시한 제품 가정

- “4마디 작은악절”은 100개 원형 진행을 4마디로 정규화한 화성 골격이다. 한 마디에 화음 하나를 기본으로 하고, 원형 보존에 필요하면 2개까지 허용한다.
- 8·12·16마디 곡은 작은악절 2·3·4개를 연결해 만든다.
- 기본 악보는 높은음자리표 단선율 한 단이다. 반주는 재생되지만 반주 전체를 피아노 대보표로 적지는 않는다.
- 조성은 C장조·A단조를 쉬운 기본값으로 두고, 교사 설정에서 12개 장·단조 조옮김을 제공한다.
- 한 마디 안의 화음 3개 이상, 중간 박자표 변경, 다성부 편집은 1차 출시 범위 밖이다.

## 4. 사용자

### 주 사용자: 초등학생 작곡가

- 7~13세, 악보 읽기 능력은 없음부터 기초 수준까지 다양하다.
- 분위기와 소리는 말할 수 있지만 화음 이름은 모를 수 있다.
- 큰 버튼, 짧은 문장, 즉시 재생, 실수 복구가 필요하다.

### 보조 사용자: 교사

- 수업용 과제를 만들고 학생의 완성 여부를 확인한다.
- 40분 수업 안에 설명·작곡·발표가 끝나야 한다.

### 보조 사용자: 보호자

- 계정과 공개 범위를 관리하고 결과물을 저장·인쇄한다.
- 만 14세 미만 사용자의 동의와 개인정보 권리를 행사한다.

## 5. 음악 이론 기반과 콘텐츠 정책

### 5.1 채택 모델

기능화성의 기본 흐름을 내부 생성 규칙으로 쓴다.

- 으뜸 기능(T): 안정과 출발·도착. 장조의 I, vi 일부와 단조의 i, VI 일부.
- 버금딸림/전딸림 기능(PD): 으뜸에서 벗어나 딸림을 준비. 장조의 ii, IV와 단조의 ii°, iv 등.
- 딸림 기능(D): 긴장을 만들고 으뜸으로 해결. V, V7, vii°.
- 기본 문장: T → PD → D → T.
- 변형: T의 연장, 대리화음, 속임종지(V→vi/VI), 플라갈 종지(IV→I), 대중음악 순환 진행을 허용한다.

이 모델은 서양 조성음악을 위한 하나의 유용한 틀이지 절대 규칙이 아니다. Open Music Theory의 phrase model도 최소한 으뜸과 딸림 기능이 진행 방향을 만든다고 설명하며, 이상적 악절은 으뜸에서 출발해 전딸림·딸림을 지나 으뜸으로 돌아온다. 출처는 [Introduction to Harmony](https://pressbooks.nebraska.edu/openmusictheory/chapter/intro-to-harmony/)와 [Harmonic Syntax](https://openmusictheory.github.io/harmonicSyntax1.html)를 따른다.

### 5.2 어린이에게 보이는 표현

아이 화면은 “이론 이름”이 아니라 “소리가 어떻게 움직이고 어떤 느낌이 드는지”를 말한다. 4~6학년이 뜻을 바로 짐작할 수 있도록 한 문구는 12자 안팎, 설명은 한 문장으로 제한한다.

| 내부·교사용 개념 | 아이 화면의 짧은 말 | 아이에게 들려주는 설명 |
|---|---|---|
| Tonic / 으뜸 기능 | 우리 집 화음 | 편안하고 제자리로 돌아온 느낌이에요. |
| Predominant / 버금딸림 | 길을 떠나는 화음 | 이야기가 다음 장면으로 움직여요. |
| Dominant / 딸림 기능 | 돌아가고 싶은 화음 | 살짝 긴장되고 우리 집 화음을 기다리게 해요. |
| Authentic cadence / 정격종지 | 집에 딱 도착 | 노래가 또렷하게 끝나요. |
| Plagal cadence / 변격종지 | 포근하게 도착 | 노래가 부드럽고 편안하게 끝나요. |
| Half cadence / 반종지 | 다음 편을 기다려 | 아직 끝나지 않은 질문처럼 들려요. |
| Deceptive cadence / 위종지 | 어? 다른 길이네 | 끝날 줄 알았는데 예상 밖으로 이어져요. |
| Major / 장조 | 맑고 밝은 색 | 햇살처럼 환하고 힘이 나요. |
| Minor / 단조 | 깊고 차분한 색 | 밤이나 비처럼 생각이 깊어져요. |
| Jazz harmony | 밤거리 화음 | 여러 음이 포개져 부드럽고 멋져요. |
| Modal harmony / 선법 | 낯선 세계 화음 | 평소와 다른 음의 색깔로 새로운 장소를 만들어요. |
| Diminished chord / 감화음 | 조마조마 화음 | 금방 어디론가 움직일 것처럼 아슬아슬해요. |
| Augmented chord / 증화음 | 점점 커지는 화음 | 소리가 쭉 늘어나며 긴장이 커져요. |
| Suspended chord / sus4 | 기다리는 화음 | 한 음이 자리를 비켜 주기 전까지 답을 기다려요. |
| Secondary dominant | 다음 화음 밀어주기 | 다음 화음으로 힘차게 넘어가게 도와줘요. |
| Tritone substitution | 반 칸 미끄럼 | 옆 칸에서 미끄러지듯 목적지에 도착해요. |
| Borrowed chord | 다른 색 빌리기 | 잠깐 다른 음의 색깔을 빌려 분위기를 바꿔요. |
| Pedal point | 아래음 붙잡기 | 아래의 한 음을 오래 잡고 위 화음만 바꿔요. |
| Line cliché | 한 칸씩 걷는 음 | 화음 속 한 음이 계단처럼 조금씩 움직여요. |
| Chromatic motion | 반음 계단 | 피아노의 바로 옆 건반으로 한 칸씩 움직여요. |
| Atonal-directed | 일부러 길 잃기 | 집처럼 느껴지는 음을 숨겨 낯설게 만들어요. |
| Pentatonic | 다섯 음 놀이터 | 다섯 음을 중심으로 단순하고 넓게 노래해요. |

학생 화면에서 금지하는 낱말은 재즈, 모달, 감화음, 증화음, 도미넌트, 서브도미넌트, 트라이톤, 크로매틱, 무조성, 메이저, 마이너, dim, aug, sus, ii–V–I 등이다. 이 용어는 교사 모드의 “이론 이름 보기”를 켰을 때만 괄호 안에 표시한다.

### 5.3 첨부 이미지 반영

첨부 예시는 4/4 한 마디 가락을 으뜸·버금딸림·딸림 화음별로 열에 나누고, 4분·8분·2분음표와 쉼표를 조합한다. 제품에서는 이 구조를 다음처럼 확장한다.

- 화음 “기능”뿐 아니라 실제 조성과 코드 구성음을 기준으로 후보를 검증한다.
- 같은 화음에도 리듬 밀도, 음역, 윤곽, 시작 쉼표, 종지감이 다른 후보를 제공한다.
- 4/4 외 5개 박자에도 박자별 강세 구조와 정확한 마디 길이를 적용한다.
- 이미지처럼 한눈에 비교하되, 각 카드에 재생 버튼과 “차분함/통통 튐/가사 많음” 태그를 붙인다.

### 5.4 멜로디 적합성 규칙

각 음은 절대음이 아니라 조성 안의 음계도수로 저장한 뒤 선택 조성에 맞춰 변환한다.

1. 강박, 복합박자의 주박, 마디 중간 화음 교체 시점의 시작음은 원칙적으로 그 시점에 활성화된 화음의 구성음이다.
2. 약박의 비화성음은 순차 진행하는 경과음·이웃음으로 허용하며 다음 강박 안에 해결한다.
3. 한 화음 위에서 증4도·장7도 도약은 기본 후보에서 제외한다.
4. 연속 도약은 같은 방향으로 2회를 넘기지 않고, 큰 도약 뒤에는 반대 방향 순차 진행을 우선한다.
5. 기본 음역은 C4~D5, 저학년 모드는 D4~B4, 넓은 음역 옵션은 A3~E5다.
6. 한 마디의 최고음·최저음 차이는 기본 한 옥타브 이내다.
7. 닫힌 4마디의 마지막 강박은 으뜸화음 구성음, 열린 4마디는 딸림음 또는 딸림화음 구성음을 우선한다.
8. 같은 음 5회 초과 반복, 직전 마디와 완전 동일한 가락, 노래하기 어려운 빠른 도약은 감점한다.
9. “우리 가락” 태그는 오음음계 선율 재료를 쓰되, 반주는 별도의 서양 기능화성 편곡임을 교사용 설명에 명시한다.

## 6. 핵심 사용자 흐름

1. 새 노래 만들기
2. 노래 분위기 듣기 및 4마디 작은악절 선택
3. 박자 선택: 2/4, 3/4, 4/4, 6/8, 9/8, 12/8
4. 곡 길이 선택: 8, 12, 16마디
5. 각 4마디 구간에 작은악절을 배치하거나 추천 조합 사용
6. 마디별 가락 카드를 듣고 하나씩 선택
7. 전체 재생 및 마디 직접 편집
8. 주선율 악기와 반주·리듬 악기 선택
9. 음표 아래에 가사 음절 배치
10. 완성 검사
11. 저장, A4 PDF, 비공개/학급/링크/공개 공유
12. 다른 사용자는 “리메이크”로 사본을 만들고 원작자 표시는 보존

가락 선택 전에 “가사 메모”를 적어 음절 수 필터를 쓸 수 있지만, 정식 음표-가사 배치는 악기 선택 후 단계에서 한다.

## 7. 기능 요구사항

### FR-01 시작과 프로젝트

- 로그인 없이 체험곡 1개를 브라우저에 임시 저장할 수 있다.
- 계정 사용자는 자동 저장, 여러 기기 동기화, 공유를 쓸 수 있다.
- 새 프로젝트 기본값은 C장조, 4/4, 8마디, 피아노, 96 BPM이다.
- 모든 편집은 실행 취소/다시 실행을 지원한다.

수용 기준:

- 네트워크가 10초 끊겨도 현재 편집은 로컬에 보존된다.
- 마지막 변경 후 2초 이내 자동 저장 상태가 표시된다.

### FR-02 작은악절 탐색

- 분위기, 밝고/깊고, 차분/힘참, 편안/궁금/조마조마, 끝남/이어짐, 쉬움/도전으로 100개를 필터링한다.
- 카드에는 어린이용 이름, 분위기, 색상, 4칸 흐름 그림, 10초 이내 미리듣기, “이 분위기와 비슷한 것”을 제공한다.
- 화음 이름과 원 진행 별명은 기본 숨김, 교사 모드에서 이론 분류·로마숫자·실제 코드명·원 진행 별명을 표시한다.
- 아이 카드 이름에는 전문용어와 영문·화음기호를 넣지 않고 “당당한 마침”, “별빛 미끄럼”, “조마조마 계단”처럼 느낌+움직임으로 이름을 붙인다.
- 열린 악절을 고르면 다음 구간에 해결 가능한 악절을 우선 추천한다.

수용 기준:

- 어떤 작은악절도 2번의 탭 안에 전체 미리듣기가 가능하다.
- 열린 악절 뒤에 호환되지 않는 종결 악절을 놓으면 막지 않고 더 자연스러운 대안을 제안한다.
- 학생 모드의 화면·음성 안내·오류 문구에서 금지 용어가 0건이어야 하며, 전문용어 사전 검사와 4~6학년 사용성 테스트를 통과해야 한다.

### FR-03 박자와 길이

- 시작 시 6개 박자를 소리와 박수 애니메이션으로 비교한다.
- 곡 전체 기본 박자는 하나이며 MVP에서 중간 박자표 변경은 지원하지 않는다.
- 길이는 8·12·16마디 중 선택하고 4마디 단위로 늘리거나 줄인다.

박자별 내부 용량과 강세:

| 박자 | 한 마디 용량(4분음표=1) | 기본 묶음 | 강세 |
|---|---:|---|---|
| 2/4 | 2 | 1+1 | 강-약 |
| 3/4 | 3 | 1+1+1 | 강-약-약 |
| 4/4 | 4 | 1+1+1+1 | 강-약-중강-약 |
| 6/8 | 3 | 3/8+3/8 | 강-약 |
| 9/8 | 4.5 | 3/8+3/8+3/8 | 강-약-약 |
| 12/8 | 6 | 3/8×4 | 강-약-중강-약 |

### FR-04 마디별 가락 후보

- 각 마디는 현재 화음 타임라인과 박자에 맞는 후보를 최소 24개 표시한다.
- 필터: 쉬운 가락, 가사 적음/보통/많음, 높은/낮은 음역, 차분/통통/길게, 쉼표로 시작.
- 카드의 재생은 현재 마디 반주와 함께 들려준다.
- 후보 선택 시 앞뒤 마디까지 이어 듣는 “연결 듣기”를 제공한다.
- 같은 리듬에 음높이만 바뀐 후보가 첫 화면을 과점하지 않도록 다양성 재정렬한다.

수용 기준:

- 모든 후보는 마디 길이 검증과 화성 적합성 점수 임계값을 통과한다.
- 동일 마디의 상위 8개 후보 중 리듬 패턴 4종 이상, 선율 윤곽 3종 이상이 있다.
- 미리듣기 입력 후 150ms 이내 시각 피드백, 캐시된 음색에서 500ms 이내 재생을 시작한다.

### FR-05 악보 편집

- 음표 선택 후 음높이 올림/내림, 음가 변경, 점음표, 붙임줄, 쉼표, 음표 추가·삭제를 지원한다.
- 드래그 외에도 탭과 키보드만으로 편집할 수 있다.
- 편집 중 남은 박 수를 마디 위 게이지로 실시간 표시한다.
- 부족: “1박이 비었어요. 쉼표로 채울까요?”
- 초과: “반 박이 넘쳤어요. 이 음을 줄이거나 다음 음을 지워 주세요.”
- 완성·공유·PDF는 모든 마디가 정확할 때만 가능하지만 임시 저장은 항상 가능하다.
- 화음 밖 음은 막지 않고 노란색으로 “조금 긴장되는 음”이라 안내하며 자동 수정 후보 3개를 제시한다.

정확성 규칙:

- duration 합 + rest 합 = meter numerator × 4/denominator.
- 교차 마디 붙임줄은 각 마디 내부 분량을 나누어 계산한다.
- n연음은 유리수로 계산하며 부동소수점 비교를 사용하지 않는다.
- 추가 음표의 시작 시점 중복, 음가 0 이하, 마디 경계 초과를 거부한다.

### FR-06 악기

- 기본 주선율 악기: 피아노.
- 서양·현대 음색: 피아노, 일렉트릭 피아노, 어쿠스틱 기타, 우쿨렐레, 글로켄슈필, 마림바, 바이올린, 첼로, 플루트, 클라리넷, 리코더, 트럼펫, 신시사이저.
- 한국 전통 선율 음색: 가야금, 거문고, 대금, 소금, 해금, 피리, 태평소, 아쟁.
- 한국 전통 리듬 레이어: 장구, 북, 꽹과리, 징. 무음과 3단계 세기 제공.
- 반주 악기는 피아노, 기타, 가야금 중 하나 또는 끔.
- 모든 샘플의 라이선스, 출처, 루프 가능 여부를 관리 콘솔에 기록한다.
- 국악기 음색은 “웹앱용 샘플 음색”으로 표시하고 실제 악기의 시김새·조율을 완전히 재현한다는 표현을 금지한다.

수용 기준:

- 악기 교체는 음표·가사·화음을 바꾸지 않는다.
- 선택 악기 음역을 벗어난 음은 자동 옥타브 이동 전 사용자에게 알린다.
- 악기 샘플 실패 시 피아노로 안전하게 대체하고 알림을 표시한다.

### FR-07 가사

- 음표 하나에 한글 한 음절을 기본 배치한다.
- 하이픈으로 다음 음절, 밑줄 또는 붙임선으로 한 음절을 여러 음에 늘인다.
- 쉼표에는 가사를 배치할 수 없다.
- 붙여넣은 가사의 음절 수와 가사 가능한 음표 수가 다르면 남거나 부족한 수를 표시한다.
- 1절을 필수 지원하고 2절은 출시 후 확장 항목으로 둔다.
- 부적절한 공개 가사는 신고·검토 대상이지만 개인/학급 초안 작성을 과도하게 차단하지 않는다.

### FR-08 재생과 완성 검사

- 마디, 4마디, 전체 재생과 카운트인, 메트로놈, BPM 60~160을 지원한다.
- 재생 위치에 맞춰 음표와 가사를 하이라이트한다.
- 완성 조건: 8/12/16마디, 모든 마디 박자 정확, 모든 마디에 가락, 제목, 주선율 악기.
- 가사는 선택 사항으로 두되 “가사 없는 연주곡”으로 명시할 수 있다.

### FR-09 PDF와 내보내기

- A4 세로 PDF 기본, 16마디는 읽기 쉬우면 2페이지까지 허용한다.
- 포함: 제목, 작곡자 표시명, 박자·조성·템포, 보표, 코드 기호(선택), 가사, 페이지 번호, 리메이크 출처.
- 인쇄 여백 12mm 이상, 음표 겹침·잘림 없음.
- PDF 생성 전 동일 렌더러로 미리보기한다.
- MusicXML 내보내기는 P1 확장 기능이며 데이터 모델은 처음부터 호환되게 설계한다. MusicXML 4.0은 박자·음표·가사·악기 정보를 표현할 수 있다. [W3C MusicXML 4.0](https://www.w3.org/2021/06/musicxml40/musicxml-reference/elements/attributes/)

수용 기준:

- Chrome·Edge·Safari 최신 2개 주요 버전에서 동일 프로젝트의 음표/가사 내용이 일치한다.
- 8·12·16마디 골든 샘플을 PDF로 렌더링해 잘림과 겹침을 자동·육안 회귀 검사한다.

### FR-10 공유

- 공개 범위: 나만 보기(기본), 링크를 아는 사람, 학급, 공개.
- 공유 카드에는 제목, 표시명, 길이, 박자, 악기, 재생, 리메이크 버튼을 표시한다.
- 검색 엔진 노출은 보호자/교사 관리 공개 계정에서만 별도 동의로 허용한다.
- 공개 프로필에 학교명, 학년·반, 실명, 위치를 요구하지 않는다.
- 신고, 차단, 공유 취소, 관리자 숨김을 지원한다.

### FR-11 리메이크와 저작 계보

- 리메이크는 원본의 새 사본이며 원본 데이터는 수정하지 않는다.
- 모든 리메이크에 “원작: 표시명 · 작품명”을 고정 표시한다.
- 사용자는 출처 블록을 삭제·숨김·수정할 수 없다.
- rootCompositionId, parentCompositionId, rootAuthorId, parentAuthorId, attributionSnapshot을 서버가 기록한다.
- 원작자가 계정을 삭제해도 “탈퇴한 원작자”와 작품 식별자는 계보에 남기되 개인정보는 제거한다.
- 원작의 공유가 취소되어도 이미 적법하게 생성된 리메이크의 출처는 유지하고, 새 리메이크 생성은 중단한다.
- PDF와 공유 페이지 모두 원작자 표시를 포함한다.

수용 기준:

- 클라이언트 요청에서 출처 필드를 빼거나 바꿔도 서버가 원본 계보로 덮어쓴다.
- 3세대 리메이크에서 바로 이전 작품과 최초 원작을 모두 추적할 수 있다.

### FR-12 계정·아동 안전

- 만 14세 미만 개인정보 동의가 필요한 경우 법정대리인 동의 및 확인 흐름을 둔다.
- 아동 고지는 쉬운 문장과 그림 요약으로 제공한다. 이는 개인정보 보호법 제22조의2의 법정대리인 동의와 아동이 이해하기 쉬운 고지 요구를 반영한다. [국가법령정보센터](https://www.law.go.kr/LSW/lsSideInfoP.do?docCls=jo&joBrNo=02&joNo=0022&lsiSeq=270351&urlMode=lsScJoRltInfoR)
- 생년월일 전체 대신 연령대 등 필요한 최소 정보만 수집하는 방안을 우선한다.
- 교사 주도 학급 계정은 학생 별칭과 일회용 참여 코드 사용을 지원한다.
- 분석 이벤트에 가사 원문과 학생 표시명을 보내지 않는다.
- 출시 전 국내 개인정보·저작권 전문 법률 검토를 완료한다.

## 8. 아이가 고르는 100가지 화음 이야기

주어진 100개 목록을 공식 화성 라이브러리로 사용한다. 아이에게는 모두 “화음 이야기”라고 부르고, 이론 분류는 내부 데이터와 교사 모드에만 둔다. 일부는 전통적인 화음 연결이고 일부는 한 음씩 움직이기, 낯선 음의 색깔, 일부러 긴장 만들기 같은 작곡 장치이므로 시스템은 서로 다른 규칙으로 가락을 만든다.

- 원형: 입력받은 진행을 보존하는 교사용·내부 표기.
- 4마디 배치: 앱에서 사용하는 정규화 결과. │는 마디 경계, →는 한 마디 안의 화음 교체다.
- 3화음 원형은 종착 화음을 한 마디 더 유지하고, 5개 이상은 음악적 분절에 따라 한 마디에 최대 2개를 넣는다.
- 화음 2개가 든 마디는 단순박자에서 기본 반 마디씩, 복합박자에서 주박 단위로 배치한다.
- 홀수 주박 박자에서는 3/4를 2+1박, 9/8을 6/8+3/8로 나누는 것을 기본으로 하며, 종지 위치에 따라 반대로 배치할 수 있다. 각 splitPattern은 프리셋 데이터에 명시한다.
- 낯설고 긴장이 큰 화음 이야기는 “도전” 난이도로 표시하며 기본 추천에서는 뒤쪽에 놓되 검색·교사 배정으로 사용할 수 있다.
- 상표·작품·아티스트를 포함한 통칭은 sourceAlias로만 저장하고, 학생 화면에는 중립적인 어린이용 이름을 사용한다.
- 학생용 이름과 교사용 이론명을 분리하고, 출시 전 음악이론 전공자가 내부 분류와 화음 철자를 검수한다.

| 기다리기와 한 칸 걷기(sus·페달·라인 클리셰) | H018, H019, H033, H055, H056 | 붙잡는 아래음은 반주 규칙으로 분리하고 기다리는 화음은 답이 나오기 전 충돌음 회피 | 움직이는 속음과 가락의 반음 충돌 검사 |
| 반 칸 미끄럼(트라이톤 대리·백도어) | H032, H053, H065 | 목적지 화음으로 미끄러지는 핵심 음을 점수화 | 같은 음 유지 또는 반음 해결 우선 |
| 낯선 세계와 다섯 음 놀이터(선법·오음계) | H076, H079~H081, H086, H093 | 지정 음 모음 안에서 중심음을 강조 | “집에 도착” 대신 중심음 회귀와 특징음 사용 |
| 힘센 두 음과 반복 놀이(파워코드·미니멀) | H090, H094 | 근음·5음과 선택한 음의 색깔을 사용 | 반복 리듬과 짧은 동기 변형 강화 |
| 일부러 길 잃기(반음계·무조성 지향) | H078, H084, H087, H092, H100 | 구간별 허용음을 따르고 긴장도를 쉬움/강함으로 표시 | 같은 음·가까운 음 연결을 우선하며 기본 추천 뒤쪽 배치 |

### 9.2 후보 생성 파이프라인

1. 입력: 조성, 박자, 현재 마디의 화음 타임라인(1~2개), 이전·다음 마디 화음, 마디 위치, 종지 유형, 난이도, 가사 음절 목표.
2. 박자별 리듬 템플릿에서 정확히 한 마디가 되는 패턴 선택.
3. 상승·하강·아치·골짜기·반복·물결 등 선율 윤곽 선택.
4. 강박과 화음 교체 시점에 해당 화음의 구성음을 배치하고 약박에 허용 비화성음을 삽입.
5. 음역·도약·반복·종지 규칙 검사.
6. 앞뒤 마디 연결 비용 계산.
7. 중복 제거 후 화성 적합성, 노래 가능성, 리듬 다양성으로 순위화.
8. 상위 후보를 MIDI/오디오 이벤트와 악보 데이터로 함께 반환.

### 9.3 출시 콘텐츠 최소량

- 박자별 수작업 검수 리듬 템플릿 48개 이상, 총 288개 이상.
- 선율 윤곽 12종 × 밀도 3단계 × 시작 방식 3종.
- 화음 기능별·종지별 골든 멜로디 마디 30개 이상.
- 각 실제 슬롯에는 생성·검수된 후보 24개 이상.
- 후보 ID는 생성 규칙 버전과 시드를 포함해 같은 프로젝트에서 재현 가능해야 한다.
- 자동 생성 결과는 음악교육 전공자 2인의 샘플링 검수를 거쳐 출시한다.

### 9.4 점수 예시

## 12. 권장 기술 구조

- 프런트엔드: TypeScript strict mode의 컴포넌트 기반 SPA/SSR 프레임워크.
- 음악 도메인: 프레임워크와 분리된 순수 TypeScript 패키지. 박자 검증, 조옮김, 화성 점수, 리메이크 불변 규칙을 담당.
- 재생: Web Audio 기반 스케줄러. Tone.js는 전역 transport와 sampler를 제공하는 후보 기술이다. [Tone.js 공식 문서](https://tonejs.github.io/)
- 악보: 단일 내부 음악 모델에서 화면 SVG와 PDF를 생성. Verovio는 SVG 렌더링과 MusicXML 입력·MIDI 출력을 지원하는 후보 기술이다. [Verovio](https://www.verovio.org/)
- 백엔드: 인증, 저장, 공유 권한, 리메이크 계보, PDF 작업 큐.
- 저장소: 관계형 DB + 악기 샘플/PDF용 객체 저장소 + CDN.
- 생성 방식: 출시 1차는 규칙 기반·결정적 생성. 외부 생성형 AI 호출은 필수 의존성으로 두지 않는다.

중요 결정은 기술 스파이크로 악보 편집 가능성, PDF 한글 폰트, 모바일 오디오 지연, 국악기 샘플 라이선스를 검증한 뒤 확정한다.

- 애니메이션 감소 설정, 자막형 박 카운트, 고대비 모드를 제공한다.
- 한 화면의 주요 선택지는 8개 안팎으로 먼저 보여주고 더보기로 확장한다.

## 20. Karpathy 원칙을 적용한 개발 가이드

지정 저장소의 네 원칙인 Think Before Coding, Simplicity First, Surgical Changes, Goal-Driven Execution을 프로젝트 규칙으로 채택한다. [multica-ai/andrej-karpathy-skills](https://github.com/multica-ai/andrej-karpathy-skills)

### 생각한 뒤 구현

- 모호한 음악 규칙과 아동 정책을 임의 해석하지 않고 가정·선택지·트레이드오프를 이슈에 기록한다.
- 구현 전에 수용 기준과 실패 예시를 먼저 합의한다.
- 악보 렌더러, 샘플 라이선스 등 불확실성이 큰 항목은 짧은 기술 검증으로 확인한다.

### 단순함 우선

- 1차는 규칙 기반 단선율+반주에 집중하고 범용 DAW 추상화를 만들지 않는다.
- 내부 음악 모델은 NoteEvent, Measure, Composition의 최소 구조로 시작한다.
- 실제 두 번째 사용 사례가 생기기 전 범용 플러그인·워크플로 엔진을 만들지 않는다.

### 필요한 곳만 변경

- 각 PR은 하나의 사용자 요구 또는 검증 가능한 기술 목표에 연결한다.
- 관련 없는 리팩터링과 포맷 변경을 섞지 않는다.
- 변경으로 생긴 미사용 코드만 정리하고 기존 별도 문제는 별도 이슈로 남긴다.

### 목표 중심 실행

모든 작업은 “기능 추가”가 아니라 검증 가능한 목표로 쓴다.

- 나쁜 예: 박자 검증 추가.
- 좋은 예: 6개 박자에서 정상 마디는 통과하고 1/8박 부족·초과 마디는 정확한 안내와 함께 실패하는 테스트를 먼저 작성한 뒤 통과시킨다.
- 나쁜 예: 리메이크 만들기.
- 좋은 예: 3세대 리메이크에서 클라이언트가 출처 필드를 변조해도 최초·직전 원작자가 서버 응답과 PDF에 보존되는 E2E 테스트를 통과시킨다.

PR 템플릿 필수 항목:

1. 해결하려는 사용자 문제
2. 명시한 가정과 제외 범위
3. 성공 기준과 테스트
4. 변경 파일과 각 변경의 이유
5. 남은 위험 또는 후속 결정

## 22. 결정이 필요한 항목

1. 서비스 운영 지역이 한국 단일인지 다국가인지.
2. 학생 개인 계정과 교사 관리 학급 계정 중 출시 우선순위.
3. 공개 공유를 최초 출시부터 허용할지, 보호자/교사 승인 계정으로 제한할지.

6. 교과 과정 연계를 위한 교사용 화성 설명과 활동지 범위.
